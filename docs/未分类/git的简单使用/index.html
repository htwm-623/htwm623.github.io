git的简单使用

查看提交记录 git log​

  1. git log

  1. git log -p 详细显示

  1. git log -oneline 简单显示(常用)

状态修改 checkout/reset/ restore​

  1. 在修改文件之后, 未提交到索引区之前, 使用git checkout恢复文件

  1. 已经提交至索引区的文件 使用 git reset HEAD 撤销提交至索引状态

  1. 使用 git restore <file> / git restore --staged <file> 分别从修改/索引区 撤销

比较修改内容 git diff​

  1. 索引区之前比较 git diff

  1. 加入索引区之后比较 git diff --cached

​

文件操作 mv/rm​

  1. 文件重命名(提交到索引之后才能使用) git mv

  1. 文件从索引区删除 git rm --cached

修改最后的提交 commit --amend​

  1. 将修改追加到最后一次 commit ，需要先将文件添加到索引区

​

返回过去版本​

  • 记得备份
  1. reset
# 返回上一次提交
git reset --hard HEAD

# 返回HEAD -1 提交
git reset --hard HEAD~

# 返回上n次
git reset --hard HEAD~n

# 返回指定 提交， 可以来回跳
# git reset --hard <commitid>
git reset --hard 40400ea
  1. reflog 查看现在位置，

合并分支​

  1. git merge

  1. git branch -d [name] 删除指定分支

分支冲突​

  1. 当不同分支修改同一文件，在合并分支时

  1. 解决合并冲突，尽量使用界面， 如 vscode

使用 tag 标签​

  1. git tag 查看所有 tag
  2. git tag v1.0.0 给当前 commit 打标签

  1. 使用 git show [tag] 查看 tag 版本信息

使用别名 alias​

  1. 将 checkout 设置别名 co

git config --global alias.co checkout
